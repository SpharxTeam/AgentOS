# CoreLoopThree — 核心运行时

**路径**: `agentos/atoms/coreloopthree/`

CoreLoopThree 是 AgentOS 的核心运行时引擎，采用创新的"三循环"架构实现智能体的感知-思考-行动闭环。该设计受心理学"双系统理论"（System 1 / System 2）启发，将快速直觉思维与慢速理性思维融入同一架构。

---

## 设计理念

CoreLoopThree 的核心思想是将智能体的运行过程建模为三个相互耦合的循环：

| 循环 | 对应系统 | 功能 | 频率 |
|------|----------|------|------|
| 认知循环 | System 2 | 环境感知、信息分析、决策规划 | 低频 |
| 执行循环 | System 1 | 快速响应、指令执行、状态反馈 | 高频 |
| 学习循环 | Meta | 经验总结、模式发现、策略优化 | 后台 |

三个循环并行运行、相互协作、持续演进，构成智能体的完整生命活动。

## 三循环详解

### 认知循环 (Cognition Loop)

认知循环是智能体的"大脑"，负责高层次的信息处理和决策。

```
输入(环境/用户) → 感知 → 理解 → 决策 → 输出(行动计划)
```

1. **感知 (Perceive)**: 收集多模态输入（文本、视觉、音频），提取关键信息
2. **理解 (Comprehend)**: 利用上下文和记忆理解当前状态，识别意图和目标
3. **决策 (Decide)**: 基于理解生成行动计划，选择最佳执行路径

### 执行循环 (Execution Loop)

执行循环是智能体的"手脚"，负责将决策转化为具体行动。

```
输入(行动计划) → 规划 → 执行 → 反馈 → 输出(执行结果)
```

1. **规划 (Plan)**: 将行动计划分解为具体的可执行步骤
2. **执行 (Execute)**: 调用相应能力（工具、API、函数）执行步骤
3. **反馈 (Feedback)**: 收集执行结果，评估效果，提供给认知循环

### 学习循环 (Learning Loop)

学习循环是智能体的"成长系统"，负责从经验中学习和优化。

```
输入(历史经验) → 收集 → 分析 → 优化 → 输出(优化策略)
```

1. **收集 (Collect)**: 记录交互历史、执行轨迹、结果评估
2. **分析 (Analyze)**: 挖掘模式、发现规律、识别改进空间
3. **优化 (Optimize)**: 更新策略、调整参数、优化行为

## 状态管理

CoreLoopThree 维护四种类型的上下文状态：

| 上下文类型 | 作用域 | 生命周期 | 用途 |
|-----------|--------|----------|------|
| Session Ctx | 会话 | 一次对话 | 当前对话的连续性和上下文 |
| Task Ctx | 任务 | 任务期间 | 当前执行任务的中间状态 |
| Global Ctx | 系统 | 持久化 | 全局配置和共享状态 |
| Memory Ctx | 记忆 | 持久化 | 跨会话的知识和记忆 |

## 事件驱动模型

CoreLoopThree 采用事件驱动架构，三循环之间通过事件进行通信：

```
┌────────────┐     ┌────────────┐     ┌────────────┐
│ Cognition  │────→│ Execution  │────→│  Learning   │
│ Loop       │ 事件│ Loop       │ 事件│  Loop       │
│            │←────│            │←────│             │
└────────────┘     └────────────┘     └────────────┘
```

- **认知 → 执行**: 行动计划事件、决策指令
- **执行 → 认知**: 执行结果、状态反馈
- **执行 → 学习**: 执行轨迹、性能数据
- **学习 → 认知**: 优化建议、策略更新

## 与相关模块的关系

- **CoreKern**: 依赖微内核的调度和服务能力
- **Syscall**: 通过标准化接口调用系统能力
- **MemoryRovol**: 管理记忆上下文的持久化和检索
- **TaskFlow**: 在执行循环中驱动 DAG 任务流
- **Frameworks**: 在执行循环中调用外部 AI 框架能力

## 使用示例

```python
# CoreLoopThree 使用示例 (伪代码)
runtime = CoreLoopThree()

# 启动运行时
runtime.start()

# 提交用户请求
task_id = runtime.submit("分析这份文档并生成摘要")

# 监控执行状态
while not runtime.is_complete(task_id):
    status = runtime.get_status(task_id)
    print(f"状态: {status}")
    time.sleep(1)

# 获取结果
result = runtime.get_result(task_id)
print(result)
```

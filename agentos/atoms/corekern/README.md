# CoreKern — 微内核

**路径**: `agentos/atoms/corekern/`

CoreKern 是 AgentOS 的 **"核中之核"** ，是系统最底层的微内核实现。它提供最小化的核心服务集合，遵循经典的微内核架构设计理念——在内核态只保留绝对必要的服务，所有其他系统服务都以用户态进程的形式运行。

---

## 设计哲学

CoreKern 的设计受到 L4 微内核和 seL4 形式化验证内核的启发，强调以下原则：

- **最小化**: 内核仅包含 IPC、内存管理、任务调度、时间服务四个原子能力
- **机制与策略分离**: 内核只提供机制，策略由用户态服务决定
- **服务化**: 所有非核心功能（文件系统、网络、设备驱动）作为用户态服务运行
- **安全性**: 通过 IPC 权限检查和能力模型确保访问控制

## 核心能力

### 1. IPC/Binder — 进程间通信

提供统一、高效的进程间通信机制，是微内核的核心通信方式。

- **Binder 协议**: 轻量级 IPC 协议，支持同步/异步调用
- **服务发现**: 基于名称的服务注册与发现机制
- **权限控制**: 调用方身份验证与权限检查
- **数据传输**: 共享内存 + 消息传递混合模式

### 2. Memory Manager — 内存管理

提供基础的内存分配和虚拟内存管理能力。

- **物理内存管理**: 页框分配与回收
- **虚拟内存**: 页表管理、地址空间隔离
- **内核堆**: 高效的内核态内存分配器
- **共享内存**: 支持 IPC 零拷贝数据传输

### 3. Scheduler — 任务调度器

负责任务/线程的调度和执行。

- **调度策略**: 优先级抢占 + 时间片轮转
- **任务状态**: 就绪、运行、阻塞、挂起
- **负载均衡**: 多核环境下的任务分布
- **实时性**: 支持硬实时和软实时任务的区分

### 4. Time Service — 时间服务

提供统一的时间基准和定时器服务。

- **系统时钟**: 高精度时间戳
- **定时器**: 单次定时、周期性定时、超时机制
- **时间同步**: 分布式时间同步基础

## 与上层模块的关系

- **Syscall 层**: 基于 CoreKern 的 IPC 和调度能力，为上层提供标准化的系统调用接口
- **CoreLoopThree**: 在 CoreKern 的调度器基础上实现三层循环运行时
- **MemoryRovol**: 利用 CoreKern 的内存管理能力实现记忆存储
- **TaskFlow**: 基于 CoreKern 的任务调度能力，提供高级任务编排

```c
// 内核初始化参数
struct kern_init_params {
    size_t    heap_size;       // 内核堆大小
    size_t    max_tasks;       // 最大任务数
    size_t    max_services;    // 最大服务数
    uint32_t  tick_hz;         // 系统时钟频率
    bool      enable_smp;      // 是否启用多核
};
```

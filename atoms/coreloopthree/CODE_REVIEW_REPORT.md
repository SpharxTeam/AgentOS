### 2.1 高风险问题

#### 问题 1: dual_model.c 硬编码 GLM-5 模型

**文件位置**: `src/cognition/coordinator/dual_model.c`  
**风险等级**: 🔴 **高**  
**问题描述**: 在双模型协同策略中硬编码了 "glm-5" 作为主模型，违反了支持用户自定义配置模型的设计原则。

**具体代码位置**:
<!-- From data intelligence emerges. by spharx -->
- 第 82 行: `models[0] = "glm-5"; // 使用GLM-5作为主模型`
- 第 112 行: `err = call_model(data->llm, "glm-5", combined_prompt, &combined_result);`

**潜在影响**:
- 用户无法配置使用其他 LLM 模型
- 违反了项目的可配置性设计原则
- 降低了系统的灵活性和可扩展性

**修复建议**:
```c
// 修复方案：使用配置的主模型而非硬编码
models[0] = data->primary_model; // 使用配置的主模型

// 修复方案：使用配置的主模型进行结果综合
err = call_model(data->llm, data->primary_model, combined_prompt, &combined_result);
```

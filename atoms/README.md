## 1. 设计哲学

atoms 是 AgentOS 的内核模块，承载整个智能体操作系统的核心运行机制。其设计遵循以下哲学：

- **微内核架构**：内核只保留最基本的机制（IPC、内存管理、任务调度），所有服务运行在用户态。
- **美学与工程**：内部结构必须像外部接口一样优美，代码即艺术品。
- **可插拔策略**：认知、规划、调度等核心算法均通过策略接口动态加载，支持运行时替换。
- **系统调用抽象**：所有对外功能通过稳定、安全的系统调用接口暴露，隐藏内核实现细节。

## 3. 核心模块职责

| 模块 | 职责 | 关键组件 |
|------|------|----------|
| **core** | 微内核基础 | IPC (Binder)、内存管理、任务调度、时间服务 |
| **coreloopthree** | 三层一体 | 认知引擎、执行引擎、记忆引擎（FFI） |
| **memoryrovol** | 记忆卷载 | L1-L4四层记忆抽象、检索、遗忘 |
| **syscall** | 系统调用 | 任务、记忆、技能、Agent等系统调用实现 |

## 4. 设计原则

1. **最小特权**：每个模块只拥有完成任务所需的最小权限。
2. **清晰分层**：模块间通过稳定接口交互，禁止循环依赖。
3. **可测试性**：每个模块必须提供单元测试。
4. **性能优先**：关键路径使用 C/C++ 实现，避免不必要的抽象开销。
5. **可观测性**：所有模块集成 OpenTelemetry 埋点。

## 5. 版本兼容性

内核 API 遵循语义化版本控制：
- **主版本号**：不兼容的 API 变更
- **次版本号**：向下兼容的功能新增
- **修订号**：向下兼容的问题修复

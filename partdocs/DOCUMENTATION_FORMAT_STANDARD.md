## 一、文档结构标准

### 1.1 标题层级规范

AgentOS 文档采用 Markdown 格式，标题层级使用规范如下：

| 层级 | Markdown语法 | 使用场景 | 示例 |
|------|-------------|---------|------|
| H1 | `# 标题` | 文档主标题（仅一个） | `# CoreLoopThree 架构详解` |
| H2 | `## 标题` | 主要章节 | `## 1. 概述` |
| H3 | `### 标题` | 子章节 | `### 1.1 设计理念` |
| H4 | `#### 标题` | 细节说明 | `#### 1.1.1 核心原则` |
| H5 | `##### 标题` | 补充说明 | `##### 注意事项` |
| H6 | `###### 标题` | 极少使用 | `###### 参考链接` |

**规范要求**：
- 每个文档只能有一个 H1 标题
- 标题层级不得跳级（如 H1 直接跳到 H3）
- 标题应简洁明了，避免过长的标题

### 1.2 文档头部标准格式

所有技术文档应包含以下头部信息：

```markdown
# 文档标题

**版本**: vX.Y.Z  
**最后更新**: YYYY-MM-DD  
**状态**: [草稿/审核中/正式发布]  
**路径**: `相对路径`  
**理论基础**: [相关理论，如适用]

---
```

**字段说明**：
- **版本**: 采用语义化版本号（MAJOR.MINOR.PATCH）
- **最后更新**: 格式为 YYYY-MM-DD
- **状态**: 草稿、审核中、正式发布
- **路径**: 文档相对于项目根目录的路径
- **理论基础**: 文档所依据的理论基础（如"工程两论"）

### 1.3 文档尾部标准格式

所有技术文档应包含以下尾部信息：

```markdown
---

**版本历史**

| 版本 | 日期 | 作者 | 变更说明 |
|------|------|------|----------|
| vX.Y.Z | YYYY-MM-DD | 作者名 | 变更描述 |

**编写者**: 作者名  
**审核者**: 审核者名  
**批准日期**: YYYY-MM-DD

---

© 2026 SPHARX Ltd. All Rights Reserved.  
"From data intelligence emerges."
```

## 二、段落与文本格式规范

### 2.1 段落格式

**规范要求**：
- 段落之间使用一个空行分隔
- 段落首行不缩进
- 每个段落应表达一个完整的主题
- 避免过长的段落（建议不超过 5-6 行）

**示例**：
```markdown
这是第一段内容，描述一个完整的主题。

这是第二段内容，描述另一个主题。段落之间使用空行分隔。
```

### 2.2 强调文本

| 格式 | Markdown语法 | 使用场景 |
|------|-------------|---------|
| **粗体** | `**文本**` | 重要概念、关键术语 |
| *斜体* | `*文本*` | 强调、引用、术语首次出现 |
| `代码` | `` `代码` `` | 代码片段、命令、文件名 |
| ~~删除线~~ | `~~文本~~` | 已废弃内容 |

**示例**：
```markdown
AgentOS 是一个**智能体操作系统**，其设计基于"工程两论"。

记忆卷载（Memory Convolution）是 MemoryRovol 系统的核心机制。

使用 `agentos_cognition_process()` 函数提交任务。
```

### 2.3 列表格式

#### 无序列表

```markdown
- 项目一
- 项目二
  - 子项目一
  - 子项目二
- 项目三
```

#### 有序列表

```markdown
1. 第一步
2. 第二步
   1. 子步骤一
   2. 子步骤二
3. 第三步
```

#### 任务列表

```markdown
- [x] 已完成任务
- [ ] 待完成任务
```

**规范要求**：
- 列表项之间不留空行
- 子列表缩进 2 个空格
- 列表项首字母小写（除非是专有名词）
- 列表项末尾不加标点（除非是完整句子）

## 三、代码块格式规范

### 3.1 代码块语法

使用三个反引号包裹代码块，并指定语言类型：

```markdown
```c
int main() {
    printf("Hello, AgentOS!\n");
    return 0;
}
```
```

### 3.2 支持的语言类型

| 语言 | 标识符 |
|------|--------|
| C | `c` |
| C++ | `cpp` |
| Python | `python` |
| JavaScript | `javascript` |
| TypeScript | `typescript` |
| Go | `go` |
| Rust | `rust` |
| Bash | `bash` |
| JSON | `json` |
| YAML | `yaml` |
| Markdown | `markdown` |

### 3.3 代码块规范

**规范要求**：
- 所有代码块必须指定语言类型
- 代码块内不使用 Tab 缩进，使用 2 或 4 个空格
- 代码块应包含必要的注释
- 避免过长的代码行（建议不超过 80 字符）

**示例**：
```c
/**
 * @brief 提交任务到认知层
 * @param plan 任务规划
 * @param task_id 输出参数，任务ID
 * @return 成功返回 AGENTOS_OK，失败返回错误码
 */
agentos_error_t agentos_cognition_process(
    const agentos_plan_t* plan,
    char** task_id);
```

## 四、表格格式规范

### 4.1 表格语法

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |
| 内容4 | 内容5 | 内容6 |
```

### 4.2 表格对齐

```markdown
| 左对齐 | 居中对齐 | 右对齐 |
|:-------|:--------:|-------:|
| 内容   | 内容     | 内容   |
```

### 4.3 表格规范

**规范要求**：
- 表头行必须与内容行分隔
- 表格应简洁明了，避免过宽
- 复杂表格考虑使用列表替代
- 表格编号格式：表-X（如：表-1）

**示例**：
```markdown
**表-1：性能基准测试结果**

| 指标 | 数值 | 测试条件 |
|------|------|---------|
| 记忆写入吞吐 | 10,000+ 条/秒 | L1 层，异步批量写入 |
| 向量检索延迟 | < 10ms | FAISS IVF1024, k=10 |
| 混合检索延迟 | < 50ms | 向量+BM25, top-100 重排序 |
```

## 五、图表格式规范

### 5.1 图表编号

所有图表应统一编号，格式如下：

- **图-X**：用于流程图、架构图等
- **表-X**：用于数据表格
- **代码-X**：用于重要代码示例

**示例**：
```markdown
**图-1：CoreLoopThree 三层架构图**

\`\`\`
┌─────────────────────────────────────┐
│         认知层 (Cognition)           │
│  意图理解 → 任务规划 → Agent 调度     │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│         行动层 (Execution)           │
│  执行引擎 → 补偿事务 → 状态追踪       │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│      记忆与进化层 (Memory)           │
│  记忆存储 → 模式挖掘 → 规则更新       │
└─────────────────────────────────────┘
\`\`\`
```

### 5.2 图表引用

在正文中引用图表时，使用以下格式：

```markdown
如图-1所示，CoreLoopThree 架构包含三层。

CoreLoopThree 的性能基准测试结果见表-1。
```

## 六、链接与引用规范

### 6.1 内部链接

使用相对路径引用项目内的其他文档：

```markdown
详见：[架构设计原则](./architectural_design_principles.md)

参考：[认知层理论](../philosophy/folder/Cognition_Theory.md)
```

### 6.2 外部链接

使用完整的 URL：

```markdown
参考：[Markdown 语法](https://www.markdownguide.org/)
```

### 6.3 引用规范

**规范要求**：
- 所有链接必须有效，不得出现死链
- 内部链接使用相对路径
- 链接文本应具有描述性
- 避免使用"点击这里"等模糊描述

## 七、特殊内容格式规范

### 7.1 注释与提示

使用引用块（Blockquote）表示注释或提示：

```markdown
> **注意**：这是一个重要提示。

> **警告**：这是一个警告信息。

> **提示**：这是一个有用的提示。
```

### 7.2 数学公式

使用 LaTeX 语法表示数学公式：

**行内公式**：
```markdown
记忆保留率公式：$R(t) = e^{-t/\tau}$
```

**块级公式**：
```markdown
$$
R(t) = e^{-t/\tau}
$$
```

### 7.3 脚注

使用脚注表示补充说明：

```markdown
AgentOS 采用微内核架构[^1]。

[^1]: 微内核架构是一种将内核职责最小化的设计模式。
```

## 九、版本控制规范

### 9.1 版本号格式

采用语义化版本号：`MAJOR.MINOR.PATCH`

- **MAJOR**：重大变更，不兼容的 API 修改
- **MINOR**：新增功能，向后兼容
- **PATCH**：问题修复，向后兼容

### 9.2 变更记录格式

```markdown
**版本历史**

| 版本 | 日期 | 作者 | 变更说明 |
|------|------|------|----------|
| v1.0.0 | 2026-03-24 | AgentOS Team | 初始版本 |
| v1.0.1 | 2026-03-25 | AgentOS Team | 修正文档引用路径 |
| v1.1.0 | 2026-03-26 | AgentOS Team | 增加案例分析章节 |
```

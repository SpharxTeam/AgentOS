## 如何成为贡献者

### 贡献方式

您可以通过以下方式成为贡献者：

1. **代码贡献**
   - 修复 Bug
   - 实现新功能
   - 性能优化
   - 代码重构

2. **文档改进**
   - 修正拼写错误
   - 完善使用说明
   - 添加示例代码
   - 翻译文档

3. **测试反馈**
   - 报告 Bug
   - 提出改进建议
   - 参与测试计划

4. **社区建设**
   - 回答问题
   - 分享经验
   - 推广项目
   - 组织活动

### 贡献流程

1. **Fork 项目**
   ```bash
   # 从 Gitee 克隆（推荐）
   git clone https://gitee.com/YOUR_USERNAME/agentos.git
   cd agentos
   
   # 或从 GitHub 克隆
   git clone https://github.com/YOUR_USERNAME/AgentOS.git
   cd AgentOS
   ```

2. **创建分支**
   ```bash
   git checkout -b feature/your-contribution
   ```

3. **进行修改**
   - 编写代码和测试
   - 确保代码符合规范
   - 通过所有测试

4. **提交 PR**
   ```bash
   git push origin feature/your-contribution
   ```

5. **代码审查**
   - 回应审查意见
   - 进行必要修改
   - 获得批准后合并

详见：[CONTRIBUTING.md](CONTRIBUTING.md)

## 认可机制

### 贡献者权益

所有贡献者将享有：

- ✅ 在 AUTHORS.md 中被列出
- ✅ 在 CHANGELOG.md 中被提及
- ✅ GitHub 贡献者图表展示
- ✅ 项目纪念品（可选）

### 贡献等级

根据贡献程度，贡献者分为：

1. **荣誉贡献者**: 1 次有效贡献
2. **活跃贡献者**: 累计 5 次有效贡献
3. **核心贡献者**: 持续贡献且质量优秀

### 晋升机制

- 每季度评估一次
- 由核心维护者评审
- 在 Release Note 中公布

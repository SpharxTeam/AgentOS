# AgentOS 客户端设计系统

## 1. 设计理念

**核心原则**：简约、清晰、直观、一致

**设计目标**：
- 精简视觉元素，突出核心功能与信息
- 建立统一的设计语言与交互模式
- 优化信息架构，降低用户认知负荷
- 提升页面响应速度与操作流畅度

## 4. 组件设计规范

### 4.1 卡片

**样式**：
- 圆角：`8px`
- 边框：`1px solid var(--border-color)`
- 背景：`var(--bg-secondary)`
- 阴影：无（简约设计）
- 内边距：`16px`

**用法**：
- 用于组织相关内容
- 作为信息展示的基本单元
- 避免过度使用卡片嵌套

### 4.2 按钮

**类型**：
- 主按钮：`--primary-color` 背景
- 次按钮：透明背景，`--border-color` 边框
- 文本按钮：无背景，无边框

**样式**：
- 圆角：`6px`
- 高度：`32px`
- 内边距：`8px 16px`
- 字体大小：`12px`
- 字重：`600`

**图标按钮**：
- 宽度：`32px`
- 高度：`32px`
- 图标大小：`16px`

### 4.3 输入框

**样式**：
- 圆角：`6px`
- 边框：`1px solid var(--border-color)`
- 背景：`var(--bg-tertiary)`
- 高度：`32px`
- 内边距：`8px 12px`
- 字体大小：`12px`

**状态**：
- 聚焦：`1px solid var(--primary-color)` 边框
- 错误：`1px solid var(--error-color)` 边框

### 4.4 表格

**样式**：
- 边框：`1px solid var(--border-color)`
- 表头背景：`var(--bg-tertiary)`
- 行高：`40px`
- 字体大小：`12px`

**交互**：
- 鼠标悬停：`--bg-tertiary` 背景
- 选中状态：`--primary-light` 背景

### 4.5 标签

**样式**：
- 圆角：`4px`
- 内边距：`2px 8px`
- 字体大小：`10px`
- 字重：`600`

**类型**：
- 主要标签：`--primary-light` 背景，`--primary-color` 文字
- 成功标签：`#10b98110` 背景，`#10b981` 文字
- 警告标签：`#f59e0b10` 背景，`#f59e0b` 文字
- 错误标签：`#ef444410` 背景，`#ef4444` 文字

### 4.6 图标

**使用规范**：
- 统一使用 Lucide React 图标库
- 图标大小：`16px`（默认）、`14px`（小）、`18px`（大）
- 图标颜色：与文本颜色保持一致
- 图标与文本间距：`8px`

## 5. 布局系统

### 5.1 页面布局

**结构**：
- 顶部导航栏：固定高度，包含品牌标识、导航链接、用户信息
- 侧边栏：固定宽度，包含主要导航菜单
- 主内容区：自适应宽度，包含页面标题、工具栏、内容区域

**间距**：
- 页面边距：`20px`
- 组件间距：`12px`
- 内容间距：`8px`

### 5.2 响应式设计

**断点**：
- 桌面端：`1200px` 以上
- 平板端：`768px` - `1199px`
- 移动端：`767px` 以下

**适配策略**：
- 桌面端：完整布局，侧边栏展开
- 平板端：侧边栏可折叠，内容区域适应
- 移动端：侧边栏隐藏（通过菜单按钮呼出），单列布局

## 6. 交互模式

### 6.1 反馈机制

**加载状态**：
- 按钮加载：显示加载图标，禁用按钮
- 页面加载：显示全局加载指示器
- 数据加载：显示局部加载指示器

**操作反馈**：
- 成功：显示成功提示，短暂自动消失
- 错误：显示错误提示，需要用户确认
- 警告：显示警告提示，需要用户注意

### 6.2 动画效果

**原则**：
- 简洁：避免过度动画
- 目的：动画应有明确的目的，如状态变化、导航过渡
- 性能：确保动画流畅，不影响页面响应速度

**类型**：
- 淡入淡出：用于模态框、通知
- 滑动：用于侧边栏、详情面板
- 微交互：用于按钮、卡片的悬停效果

### 6.3 导航模式

**顶部导航**：
- 品牌标识：左侧
- 主要导航：中间
- 用户信息：右侧

**侧边栏导航**：
- 分组显示：按功能类别分组
- 当前页面：高亮显示
- 折叠功能：适应小屏幕

**面包屑导航**：
- 显示当前页面路径
- 支持快速返回上级页面

## 7. 页面设计规范

### 7.1 通用页面结构

1. **页面头部**：
   - 页面标题：`--font-size-2xl`，`--font-weight-bold`
   - 页面描述：`--font-size-md`，`--text-secondary`
   - 操作按钮：右侧对齐

2. **工具栏**：
   - 搜索框：左侧
   - 筛选器：中间
   - 操作按钮：右侧

3. **内容区域**：
   - 数据展示：表格或卡片网格
   - 分页：底部

4. **详情面板**：
   - 从右侧滑入
   - 包含详细信息和操作

### 7.2 特定页面规范

**Dashboard**：
- 概览卡片：网格布局
- 关键指标：突出显示
- 快速操作：便捷访问

**Agents**：
- 智能体卡片：包含名称、类型、状态
- 批量操作：支持多选
- 详情面板：显示详细信息

**Tasks**：
- 任务列表：表格展示
- 状态筛选：快速过滤
- 任务详情：侧边面板

**Tool Manager**：
- 工具列表：卡片网格
- 分类筛选：按类型过滤
- 配置界面：模态框

**Terminal**：
- 命令输入：固定底部
- 输出区域：滚动显示
- 历史记录：可查看

**Settings**：
- 配置项：分组显示
- 表单元素：简洁清晰
- 保存操作：明确可见

## 8. 实现指南

### 8.1 组件复用

- 提取通用组件：按钮、输入框、卡片、表格等
- 建立组件库：确保一致性和可维护性
- 文档化组件：提供使用示例和API

### 8.2 性能优化

- 代码分割：按需加载页面组件
- 虚拟滚动：处理大量数据列表
- 缓存策略：减少重复请求
- 优化渲染：避免不必要的重渲染

### 8.3 可访问性

- 语义化HTML：使用正确的标签
- 键盘导航：支持键盘操作
- 屏幕阅读器：提供适当的ARIA属性
- 颜色对比度：确保文本可读

### 8.4 国际化

- 统一的i18n实现
- 支持多语言切换
- 文本提取：便于翻译
- 日期、时间、数字格式：根据地区适配

## 11. 结论

本设计系统旨在为 AgentOS 客户端提供统一、简约、直观的用户界面。通过遵循这些规范，我们可以确保所有页面都具有一致的设计语言和用户体验，从而降低用户认知负荷，提高操作效率。

设计系统是一个不断演进的过程，我们将根据用户反馈和业务需求持续优化和完善。
